# Оформление интерфейса: адаптация ISA-101

Ветка `isa101-ui`. Откат к прежнему виду — `git checkout review-fixes`.

## Что это и чем это не является

ANSI/ISA-101.01-2015 — стандарт про **экраны оперативного управления процессом**:
мнемосхемы, на которых оператор принимает решения в реальном времени. OptiCIP
Dashboard — журнал моек и аналитика, а не оперативный экран. Поэтому здесь
применены **принципы** стандарта, а не его буква:

- цвет несёт смысл, а не украшает;
- нормальное состояние не кодируется цветом;
- аварийные оттенки не используются больше нигде;
- никакой имитации объёма (тени, градиенты, блики).

Формально соответствовать ISA-101 приложение не обязано и не заявляет этого.

Источник палитры — `/home/Urevich/Elems/STYLE.md` (HMI Style Guide библиотеки
мнемосхемы, база — Rockwell PlantPAx PROCES-WP023). Оба интерфейса теперь
опираются на одну палитру, поэтому дашборд и панель оператора выглядят как
одна система.

## Палитра

Единственный источник истины — блок `:root` в `webapp/static/style.css`.

| Токен | HEX | Назначение |
|---|---|---|
| `--bg` | `#e0e0e0` | фон приложения |
| `--panel` / `--panel-strong` | `#e8e8e8` / `#ededed` | панели, поля ввода |
| `--surface-active` | `#c0c0c0` | нажатое / выбранное состояние |
| `--line` / `--line-soft` | `#a0a0a4` / `#c4c4c8` | границы |
| `--text` / `--muted` / `--dim` | `#3f3f3f` / `#6e6e70` / `#919191` | текст |
| `--on` | `#6f9f82` | включено (только тумблеры) |
| `--alarm` / `--alarm-dark` / `--alarm-bg` | `#c0392b` / `#8e2a20` / `#ecd9d6` | **только аварийный результат мойки** |
| `--manual` | `#356a9c` | переходное состояние (прогресс обработки) |
| `--accent` | `#356a9c` | фокус и выделение — **аффорданс интерфейса, не состояние процесса** |

## Ключевые решения

**Результат мойки.** Было: «Завершено» — зелёный бэйдж, всё остальное — красный.
Стало: норма — нейтральный серый бэйдж (норма не должна кричать), отклонение —
красная рамка 2 px + **квадратный маркер**. Форма дублирует цвет: требование
ISA-101 «цвет не может быть единственным носителем информации» и условие
читаемости при дальтонизме. Красный на экране теперь ровно один — тот, что нужен.

**Кнопки.** Цветных кнопок нет (STYLE.md §5). Основная отличается от вторичной
**формой** — рамка 2 px против 1 px. Выбранный фильтр — заливка темнее фона плюс
жирный текст, а не синяя плашка.

**Кривые графика.** Красный `#dc2626` для температуры возврата убран: на экране,
где красный означает аварию, красная кривая нормального тренда читается как
проблема. Палитра переведена на оттенки сред из STYLE.md — синий `#2f5c8a`,
бирюзовый `#3e7c8c` (пунктир), коричневый `#8a7a6a`, приглушённый зелёный
`#6f9f82`. Красный, оранжевый, жёлтый и фиолетовый закреплены за приоритетами
аварий и в кривых не появляются — это закреплено тестом
`test_chart_style_defaults`.

**Полосы фаз.** Пастельные оранжевый/розовый/красный заменены на бледные
малонасыщенные тона: полосы — это фон и контекст, они не должны соперничать с
кривыми. Фазы подписаны текстом, так что цвет здесь не единственный признак.

**Плоскость.** Убраны все 35 теней, 14 градиентов, 5 размытий фона и скругления
как декор (углы 2 px). Мерцание скелетонов загрузки снято: по HP-HMI
единственная допустимая анимация во всей системе — мигание неквитированной
аварии, и тратить её на заглушку загрузки нельзя.

## Осознанные отклонения

Оба зафиксированы явно — тогда это решение style guide, а не недосмотр.

1. **Синий `--accent` для фокуса и ссылок.** Канон HP-HMI отдаёт синий под
   «ручной режим». Здесь синий помечает *интерактивные* элементы (фокус поля,
   ссылка), а не состояние процесса. Оправдание: приложение запускается на
   офисном ПК, а не на панели оператора, и полностью серый интерфейс без
   индикации фокуса недоступен для клавиатурной навигации.

2. **Зелёный `--on` для включённого тумблера настроек.** То же отклонение, что
   принято в библиотеке мнемосхемы (STYLE.md, отклонение 1). Компенсация та же:
   рядом стоит подпись, состояние дублируется словом.

## Печать и PDF

Печатные правила (`.chart-print-*`, `body.modal-export-mode`) намеренно **не
переведены** на серый фон: бумага остаётся белой, меняются только чернила
(нейтральные серые вместо зеленоватых). Фон дисплея `#e0e0e0` в печать не идёт.

## Проверка

```bash
# запрещённое HP-HMI не должно встречаться в стилях
grep -nE 'linear-gradient|radial-gradient|box-shadow: 0|backdrop-filter: blur' webapp/static/style.css

# аварийные оттенки — только в токенах --alarm*
grep -nE '#(dc2626|ef4444|f97316|eab308|e22028)' webapp/ wash_report.py -r
```

Оба должны быть пустыми.
