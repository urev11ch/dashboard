# OptiCIP Dashboard — карта возможностей

Отчёты по мойкам CIP: приложение читает данные панелей Weintek (файлы `.db`),
строит журнал моек и графики параметров (температура подачи/возврата,
концентрация, расход). Работает локально на ПК.

> Этот файл — сводка возможностей и архитектуры, чтобы быстро войти в контекст
> без изучения всего кода. Обновлять при добавлении фич.

## Режимы запуска
- **Десктоп** (`run_wash_desktop.py`): локальный FastAPI-сервер (uvicorn на
  127.0.0.1) в окне **pywebview + WebView2** (`gui="edgechromium"`). Кастомное
  **frameless-окно** со своей шапкой.
- **Веб** (`run_wash_ui.py` / `start.sh`): тот же интерфейс в браузере, без
  нативного окна (титлбар/автозапуск/трей недоступны).
- **Сборка**: PyInstaller (`OptiCIP-Dashboard.spec`) + Inno Setup
  (`installer.iss`) на Windows через GitHub Actions
  (`.github/workflows/windows-build.yml`). Артефакт: `OptiCIP-Dashboard-Setup.exe`.
- **Релиз**: пуш тега `vX.Y.Z` → та же сборка + публикация GitHub Release с
  установщиком и `.exe`. Тег сверяется с `webapp/__init__.py` **до** сборки:
  расхождение роняет прогон. Публикуется сразу боевым релизом (не draft и не
  prerelease) — `releases/latest`, который читает проверка обновлений, их
  пропускает.

## Источники данных
- **FTP-панели**: скачивание архивов/`.db` в зеркало `datalog/<id>/`, инкрементально
  (пропуск неизменившихся по размеру/времени). Несколько сохранённых панелей,
  пароли защищены: DPAPI на Windows, системное хранилище секретов (Secret
  Service/Keychain через `keyring`) на Linux/macOS, с фолбэком на base64.
  Новые файлы кладутся помесячно: `datalog/<id>/ГГГГ-ММ/…` (месяц — по времени файла).
- **Папка/архивы**: локальная папка с `.db` и архивами `.zip`/`.tar.*`.
  Путь по умолчанию и последний открытый путь запоминаются.

## Поток данных
FTP/папка → распаковка архивов (кэш) → анализ `.db` (`wash_report.py`,
инкрементальный кэш) → журнал моек + данные графиков (кэш payload).
Сэмплы временных рядов **не держатся в RAM**: выносятся в side-файлы кэша по
потокам (`ws-samples-*`) и подтягиваются лениво (LRU) при построении графика/
оценке концентрации — в памяти остаются только метаданные и агрегаты.

## Тесты
- **Python** (`pytest`, ~151): ядро анализа, кэши, безопасность, настройки, FTP.
- **JS unit** (`vitest`, `tests/js/`): чистые функции графика (`npm run test:unit`).
- **E2E** (`playwright`, `tests/e2e/`): открытие мойки/графика, сортировка,
  разблокировка кнопки при сбое обновления (`npm run test:e2e`, системный Chrome).

## Возможности (frontend)
- **Экран выбора источника** — минималистичный (FTP / Папка), сохранённые FTP-панели.
- **Список моек** — виртуализированный; колонки: дата/время, объект, программа,
  длительность, результат. **Группировка по дням** (заголовки-разделители) при
  сортировке по дате. Поиск, фильтр по периоду/каналу, сортировка. **Скелетоны**
  при первой загрузке.
- **Результат мойки** — статус красится по категории: `completed` → зелёный,
  `check` → красный (независимо от текста подписи). Тумблер **«Требовать шаг
  окончания мойки»** (`require_completion_step`, по умолчанию выключен): при
  включении мойка без финального шага «Окончание мойки» (process 21) понижается
  до «Требует проверки». Применяется на чтении — смена действует сразу.
- **Оценка по нормативам концентрации** — концентрация раствора (щёлочь/кислота)
  сравнивается с вручную заданным нормативом (± допуск %). Датчик на возврате, поэтому
  оценивается **рабочая полка**: логика находит участок выхода на режим (первое→последнее
  превышение порога) и ловит два дефекта — **недостижение** (раствор не поднялся до нормы)
  и **провал** (просел на рабочем участке). Переходные края (заполнение контура в начале,
  вытеснение в конце) не учитываются. Ниже нормы → мойка «Концентрация ниже нормы»
  (категория `check`) + чип «конц. ↓»; в деталях/PDF — «макс/мин X% / норма Y%».
  Включается тумблером.
- **Окно графика** (модалка) — SVG-график по каналам, сводка, настройка кривых,
  навигация между мойками, **PDF/печать**. В **уменьшённом окне** показывается
  только график (без сводки/параметров) + компактная подпись объект/мойка
  оверлеем; в развёрнутом — со сводкой. График **вписывается** в область.
- **Редактор объектов** — переименование по паре канал:object_id
  (`wash_object_names.json`), поля выровнены, кнопка «Добавить объект» внизу.
- **Окно настроек** — в стиле System Settings macOS: слева боковая навигация
  разделов (активный синим, подсветка при наведении), справа контент. Разделы:
  Общие (результат мойки + нормативы концентрации + источник данных), FTP,
  График (цвета/линии + **очистка кэша графиков**), Обновления и автозапуск,
  Архивы, Диагностика.
- **Диагностика** — раздел внутри настроек (не отдельное окно): тип источника,
  путь, последняя синхронизация, счётчики, автообновление, объём `datalog`,
  хранение/последняя очистка, ошибки. Грузится при открытии раздела.
- **Тосты** — всплывающие уведомления (успех/ошибка/инфо).
- **Часы** — текущая дата и время в шапке.
- **Кастомное окно** — свернуть/развернуть/закрыть, перетаскивание за шапку;
  два режима: развёрнуто ↔ уменьшённое по центру экрана (нативно через WinForms).

## Возможности (функционал)
- **Автообновление с FTP в фоне** — периодически (интервал в настройках, по
  умолчанию 5 мин), без блокирующего оверлея.
- **Проверка обновлений** — тумблер (по умолчанию выключен). При запуске сверяет
  `webapp.__version__` с `tag_name` последнего релиза (`GET /api/update-check` →
  `api.github.com/repos/urev11ch/dashboard/releases/latest`); при наличии более
  новой версии показывает тост. Сравнение — покомпонентное по числам из тега
  (`_parse_version`), префикс `v` срезается. Сеть недоступна или релизов нет →
  молча ничего не показывает.
- **Установка обновления в один клик** — кнопка в настройках (раздел
  «Обновления») при доступном обновлении. Приложение скачивает
  `OptiCIP-Dashboard-Setup.exe` с прогрессом (`POST /api/update/download`,
  опрос `GET /api/update/job`), **сверяет sha256 с digest из ответа GitHub**,
  запускает установщик с `/SILENT /RELAUNCH=1` и закрывается; установщик ждёт
  освобождения `AppMutex`, ставит новую версию и поднимает приложение обратно.
  Windows покажет запрос UAC (установка в Program Files, `PrivilegesRequired=admin`).
  Доступно только в десктоп-сборке под Windows — `installable` в `/api/update-check`
  плюс наличие `pywebview.api.install_update`; в браузере панель даёт ссылку на Releases.
  **Защита:** URL берётся только из ответа GitHub и обязан начинаться с
  `UPDATE_ASSET_URL_PREFIX`; вложение без корректного `sha256` игнорируется;
  файл кладётся в приватный `0700`-каталог; `DesktopBridge.install_update()`
  **не принимает путь от JS** (иначе любой скрипт в окне исполнил бы произвольный
  `.exe` с правами администратора) — берёт его из состояния сервера, где `status`
  становится `ready` только после сверки суммы.
- **Автозапуск с Windows** — тумблер, ключ реестра `HKCU\...\Run`.
- **Хранение архивов** — помесячная раскладка + автоочистка старше N дней
  (1–730, по умолчанию 365, по умолчанию **выключено**). Не качает файлы старше
  срока. Кнопка «Очистить сейчас». «Последняя очистка» отмечается только при
  реальном удалении.

## Хранилище (per-user, рядом с приложением)
- `datalog/` — скачанные архивы/`.db` (FTP-зеркала `datalog/<id>/ГГГГ-ММ/`).
- `temp/` — настройки: `wash_app_settings.json`, `wash_chart_styles.json`,
  `wash_folder_source.json`, `wash_ftp_sources.json`, `wash_object_names.json`.
- Кэши анализа/архивов/графиков — очищаются при выходе.

## HTTP API (FastAPI)
- Источник: `POST /workspace/open`, `/workspace/open-ftp`,
  `/workspace/ftp-source/delete`, `/workspace/reset`, `/workspace/refresh`,
  `POST /api/workspace/refresh`.
- Данные/джоб: `GET /api/workspace-data`, `/api/workspace-job`,
  `/api/workspace-job/stream` (SSE), `POST /api/workspace-job/cancel`.
- Детали/график: `GET /api/wash-details`, `/api/wash-chart-data`.
- Объекты: `POST /api/object-name`, `/api/object-names-file/sync`.
- Настройки/стили: `GET|POST /api/settings`, `GET|POST /api/chart-styles`.
- Сервис: `GET /api/diagnostics`, `GET /api/update-check`,
  `POST /api/update/download`, `GET /api/update/job`,
  `POST /api/archives/cleanup`.

## Ключевые файлы
- `wash_report.py` — ядро анализа `.db`, результат мойки, форматирование.
- `webapp/app.py` — FastAPI: источники, FTP, анализ, кэши, настройки, API.

## Не сделано / оговорки
- Оценка по нормативам реализована для **концентрации** (щёлочь/кислота). Пороги
  по **температуре/времени выдержки** фаз — пока нет (возможное развитие).
- Норматив концентрации **глобальный**: применяется ко всем мойкам, включая старые
  (снятые при другой заправке станции). Отражает «текущую» уставку.
- «Свернуть в трей» — не реализовано (нужна зависимость `pystray`).
- Установщик **не подписан**: шаги `Sign executable`/`Sign installer` в CI ждут
  секрет `WINDOWS_PFX_BASE64` (`if: env.HAS_SIGNING == 'true'`). Без него Windows
  показывает предупреждение SmartScreen, а UAC при обновлении — «Издатель:
  неизвестен». Подпись убрала бы и то, и другое (UAC остался бы, но «зелёный»).
- Обновление «в один клик» всё же требует **одного подтверждения UAC**: установка
  идёт в Program Files. Убрать его можно только переездом на пользовательскую
  установку (`{localappdata}` + `PrivilegesRequired=lowest`) — тогда нужна
  миграция уже установленных копий.
- Обновление в **веб-режиме** не ставится (нет моста и нативного окна) — панель
  показывает ссылку на Releases.
